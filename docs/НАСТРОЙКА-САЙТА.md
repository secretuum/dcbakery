# DC Bakery — как настраивать и регулировать сайт

Короткий справочник для владельца: что где меняется, что для этого нужно и какие команды выполнять.

---

## 1. Тексты, контакты и график — БЕЗ программиста

### Суперадмин и режим редактирования

Суперадмин — это админ, которому доступно редактирование контента сайта.

- По умолчанию суперадминами считаются **все админы**.
- Чтобы ограничить круг, добавьте на Render переменную `SUPERADMIN_EMAILS` со списком
  email через запятую (например `room@secretuum.com`). Остальные админы продолжат
  работать с заказами, но редактирование контента не увидят.

**Два способа менять контент:**

1. **Карандашиком прямо на сайте**: войдите в админку, откройте главную страницу сайта —
   в правом нижнем углу появится кнопка **«Редактировать сайт»**. Включите её — у
   редактируемых текстов появятся карандашики ✎. Кликаете, меняете, «Сохранить» —
   изменение сразу на сайте.
2. **Формой в админке**: Админка → Настройки → блок **«Контент сайта»**.

**Что редактируется** (хранится в Supabase, таблица `app_settings`, ключ `site_content`):

| Поле | Где показывается |
|---|---|
| WhatsApp (бот заявок) | подвал сайта, страница «Контакты» |
| Телефон | подвал, «Контакты» |
| Адрес | подвал, «Контакты» |
| Режим работы | подвал |
| Дни доставки + час отсечки | подвал («График поставок») и календарь в оформлении заказа |
| Заголовок и подзаголовок главной | первый экран |
| Заголовок и текст «О компании» | главная |

### График поставок

Дни доставки (например Вт·Чт·Сб) и «приём заявок до 18:00» задаются в том же блоке
«Контент сайта». Календарь в оформлении заказа автоматически:
- даёт выбрать **только** дни доставки;
- закрывает ближайший день, если время отсечки накануне уже прошло.

## 2. Товары и цены

- **Цена, остаток, название, описание, фото, статус (активен/архив)** — Админка → Товары.
  Правки сохраняются в Supabase и сразу видны на сайте (перекрывают код).
- **Базовый каталог** живёт в коде: `src/data/products.ts` (цены из КП от 17.07.2026).
- **Фото** лежат в `public/products/<slug>.webp`. Заменить фото можно из админки
  (загрузка в карточке товара) или положив файл с тем же именем в эту папку.
- **Лимиты в админке**: цена до 20 000 ₸, остаток до 100.

### Скрытые категории (готовые обеды, банкетные десерты)

Категории подготовлены, но спрятаны с витрины. В админке они видны и редактируются.
Включение — переменные окружения на Render (Environment → Add):

| Категория | Переменная |
|---|---|
| Готовые обеды (ланчи 1–30) | `NEXT_PUBLIC_READY_MEALS=1` |
| Банкетные десерты | `NEXT_PUBLIC_BANQUET_DESSERTS=1` |

После добавления переменной Render сам переразвернёт сайт, категория появится на витрине.
У ланчей 19–30 нет составов и фото (их нет ни в КП, ни в iiko) — заполните в админке.

### Популярность

«Популярное» и сортировка «По популярности» считаются **автоматически по реальным
заказам** (сколько единиц товара заказали). Данные обновляются раз в 30 минут.
Ручной ранг «Популярность 1–15» в админке работает как добавка: он решает, кто выше
при равном количестве заказов, и двигает новинки, которые ещё не заказывали.

## 3. Акции

Акции на главной задаются в файле `src/data/promotions.ts` — одна запись на акцию
(заголовок, условия, подарки, период). Добавили запись → закоммитили → задеплоили.
Текущая акция: «от 100 000 ₸ в неделю → 5 десертов в подарок», 01.08–30.11.2026.

---

## 4. Что задаётся переменными окружения (Render → Environment)

| Переменная | Что делает |
|---|---|
| `ADMIN_EMAILS` | кто может войти в админку (email через запятую) |
| `SUPERADMIN_EMAILS` | кто видит редактирование контента (пусто = все админы) |
| `NEXT_PUBLIC_READY_MEALS=1` | показать «Готовые обеды» |
| `NEXT_PUBLIC_BANQUET_DESSERTS=1` | показать «Банкетные десерты» |
| `PAYMENT_MODE=manual` | оплата по счёту (реквизиты) |
| `DC_*` | реквизиты для счетов/накладных (два IBAN по цехам) |
| `NEXT_PUBLIC_WHATSAPP_SUPPORT` | номер WhatsApp-бота для кнопок «Написать» |
| `OPENAI_API_KEY` | генерация описаний из ТТК (конвейер iiko) |
| `IIKO_BASE_URL`, `IIKO_RESTO_LOGIN`, `IIKO_RESTO_PASS` | доступ к iiko (зонды/сверка) |

## 5. iiko

- Сверка цен: `node scripts/iiko-price-probe.mjs` (выгрузка) и Excel-отчёт
  (генерируется по запросу). Каналы цен в iiko: «ОПТ …» и «… дс» — прайс сайта,
  «К …» — продажи b2b-подразделения, «Каспи …» — розница.
- Выгрузка заказов сайта в iiko расходными накладными — спроектирована, ждёт решения
  по складу-источнику и тестовой накладной.

## 6. Git и деплой — правильные команды

Выполнять в папке `site`:

```powershell
git add -A                        # добавить все изменения
git commit -m "что сделано"      # зафиксировать
git push                          # отправить на GitHub
```

- Если push отклонён (кто-то пушил до вас): `git pull --rebase`, затем снова `git push`.
- Посмотреть, что незакоммичено: `git status`.
- Render деплоит автоматически после push (если включён Auto-Deploy). Вручную:
  Render → сервис → **Manual Deploy → Deploy latest commit**.
- **Если после деплоя не видно изменений**: проверьте, что коммит запушен
  (`git log --oneline -1` и сравните с Render), затем обновите страницу с очисткой
  кэша — `Ctrl+F5` (на телефоне — открыть в приватной вкладке).

## 7. Переводы (казахский/английский) — план

Полный перевод интерфейса и карточек товаров — отдельный этап:

1. Инфраструктура: словари строк (ru/kk/en), переключатель языка в шапке,
   сохранение выбора.
2. Перевод интерфейса (кнопки, формы, статусы, письма).
3. Контент товаров: составы на казахском уже есть у большинства позиций
   (`compositionKz`); названия и описания kk/en — сгенерируем конвейером
   (OpenAI по ТТК из iiko) и вычитаем.
4. SEO: hreflang, переведённые метатеги.

Оценка: это самая большая задача из списка, делается после запуска остального.
