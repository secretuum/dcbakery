# DC Bakery - текущий статус работ

Дата: 2026-06-22

## Что уже сделано

- Собран B2B-интерфейс DC Bakery на Next.js, TypeScript и Tailwind.
- Добавлены главная, каталог, карточки товаров, корзина, checkout и страница успешной заявки.
- Товары пока локальные, без Supabase-таблицы products.
- Заказы сохраняются в Supabase в таблицы `orders` и `order_items`.
- Уведомления о заказах отправляются в Telegram и WhatsApp через Green API.
- В админке есть список заказов, карточка заказа, смена статуса, подтверждение заявки и отметка оплаты.
- Добавлен ручной payment-flow: менеджер подтверждает заявку, клиент получает ссылку `/pay/[orderId]`, после оплаты заказ можно отметить как оплаченный.
- WhatsApp webhook принимает команды менеджера:
  - `подтвердить`
  - `оплачено`
  - `статус`
- Старые WhatsApp-сообщения по заказу заменяются актуальным сообщением, если Green API позволяет удалить прошлое.
- Добавлен пункт `Профиль` в шапку.
- Профиль разделён на admin/client:
  - admin открывается через Supabase Auth;
  - client пока MVP на localStorage.
- В клиентском профиле добавлена история заказов по email/телефону через `/api/profile/orders`.
- Проект задеплоен на Netlify: `https://dlcbakery.netlify.app/`.

## Что сделано в текущем шаге

- Начата операционная система ответственных лиц.
- Добавлен модуль `src/lib/responsibles.ts`.
- Добавлены роли:
  - `manager`
  - `dessertLead`
  - `semiFinishedLead`
  - `meatLead`
- Добавлены правила по категориям:
  - десерты -> менеджер + руководитель десертного цеха
  - полуфабрикаты -> менеджер + руководитель полуфабрикатного цеха
  - мясо -> менеджер + ответственный за мясо
  - смешанный заказ -> менеджер + все нужные руководители направлений
- WhatsApp-сообщение о новой заявке теперь получает блок:
  - категории заказа;
  - ответственные лица;
  - fallback-упоминания текстом через `@Имя`.
- Так как Green API `sendMessage` не дает стабильного поля для настоящих WhatsApp mentions,
  заявка дополнительно дублируется ответственным напрямую в личный WhatsApp.
- Добавлен ручной endpoint операционных напоминаний:
  - `GET /api/reminders/operations?secret=...`
  - `GET /api/reminders/operations?secret=...&dryRun=1`
  - `POST /api/reminders/operations` с заголовком `x-reminder-secret`.
- Напоминание отправляет в рабочий WhatsApp-чат сводку по ответственным, остаткам, низким остаткам,
  нулевым остаткам и актуальности каталога.

## Почему это важнее следующего шага профиля

Профиль клиента уже имеет MVP-историю заказов. Сейчас больше пользы даёт операционная часть: чтобы новая заявка сразу попадала к нужным ответственным в WhatsApp-чате и не терялась между менеджером, цехами и направлениями.

## Что важно помнить

- В текущей публичной документации Green API `sendMessage` нет отдельного стабильного поля для полноценного WhatsApp mention.
- Поэтому сейчас используется рабочий fallback для группы: номер выводится как `@7XXXXXXXXXX`.
- На практике для WhatsApp-группы полезнее именно номер без `+`, а не технический `@c.us`.
- Для надежного уведомления используется дополнительное личное сообщение ответственному.
- Если Green API авторизован на том же WhatsApp-номере, который указан ответственным, WhatsApp может
  не показать пуш-уведомление, потому что сообщение отправлено как будто с этого же аккаунта.
  Для полноценного уведомления лучше использовать отдельный WhatsApp Business номер бота.
- Для личной отправки бот сам превращает телефон в технический формат WhatsApp API.
- Реальные имена и телефоны ответственных нужно вынести в env на Netlify.

## Env-переменные для ответственных

Пока ответственный один, достаточно заполнить только общие переменные:

```text
DC_RESPONSIBLE_DEFAULT_NAME=Ответственный DC Bakery
DC_RESPONSIBLE_DEFAULT_PHONE=+7XXXXXXXXXX
```

Где это указать на Netlify:

```text
Netlify -> нужный сайт -> Site configuration -> Environment variables -> Add variable
```

Для отметки в группе используется номер без плюса:

```text
@7XXXXXXXXXX
```

Когда появятся разные ответственные по направлениям, можно заполнить отдельные переменные:

```text
DC_RESPONSIBLE_MANAGER_NAME=
DC_RESPONSIBLE_MANAGER_PHONE=

DC_RESPONSIBLE_DESSERT_LEAD_NAME=
DC_RESPONSIBLE_DESSERT_LEAD_PHONE=

DC_RESPONSIBLE_SEMI_FINISHED_LEAD_NAME=
DC_RESPONSIBLE_SEMI_FINISHED_LEAD_PHONE=

DC_RESPONSIBLE_MEAT_LEAD_NAME=
DC_RESPONSIBLE_MEAT_LEAD_PHONE=
```

## Env-переменные для напоминаний

```text
OPERATIONS_REMINDER_SECRET=сложная_секретная_строка
OPERATIONS_LOW_STOCK_THRESHOLD=5
```

Время утренней/вечерней отправки пока не добавлять в Netlify env. Его подключим на cron-шаге,
иначе Netlify secret scanner может считать обычные строки времени секретами.

Ручная проверка без отправки:

```text
https://dlcbakery.netlify.app/api/reminders/operations?secret=ВАШ_СЕКРЕТ&dryRun=1
```

Ручная отправка в WhatsApp:

```text
https://dlcbakery.netlify.app/api/reminders/operations?secret=ВАШ_СЕКРЕТ
```

Важно: секрет нельзя отправлять клиентам и нельзя публиковать в GitHub.

## Что делать дальше

1. Проверить сборку и отправку WhatsApp-сообщения с блоком ответственных.
2. Заполнить env-переменные ответственных и напоминаний на Netlify.
3. Проверить dry-run напоминания через `/api/reminders/operations?dryRun=1`.
4. Проверить ручную отправку напоминания в WhatsApp.
5. Подключить расписание через внешний cron или Netlify Scheduled Functions.
6. Добавить таблицу логов отправки напоминаний, чтобы не было дублей.
7. Добавить в админку страницу/секцию настроек ответственных, чтобы менять их без кода.
8. Доработать остатки и стоп-листы:
   - `available`
   - `low_stock`
   - `out_of_stock`
   - `stop_list`
   - `hidden`
9. После этого делать Excel-выгрузку:
   - сначала управленческий отчёт по продажам;
   - потом ABC-анализ;
   - полноценный P&L только после появления себестоимости.
10. Позже усилить клиентский профиль:
   - нормальная авторизация клиента;
   - роли admin/client;
   - повтор заказа из истории;
   - документы и счета.
