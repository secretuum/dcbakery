# Загрузка заказов сайта в 1С — сборка и подключение обработки

Обработка сама забирает заказы с сайта (исходящее HTTPS-соединение) и создаёт
в 1С контрагентов и непроведённые «Счета на оплату покупателю». На сервере 1С
ничего публиковать и настраивать не нужно.

## Шаг 0. Токен на сайте (делается один раз)

1. Придумайте/сгенерируйте длинную случайную строку (30+ символов).
2. Render → сервис сайта → Environment → добавьте `ONEC_EXPORT_TOKEN=<строка>`.
   Render передеплоит сайт автоматически.
3. Проверка из браузера не выйдет (нужен заголовок) — проверим кнопкой из 1С.

## Шаг 1. Собрать обработку (10–15 минут, один раз)

Понадобится **Конфигуратор** (ярлык 1С → выбрать базу → «Конфигуратор»).
Если входа в конфигуратор нет — это единственное, для чего понадобится помощь
(любой знакомый 1С-ник соберёт по этой инструкции за 10 минут).

1. Меню **Файл → Новый → Внешняя обработка**. Имя: `ЗагрузкаЗаказовDCBakery`.
2. В дереве обработки правой кнопкой по **Формы → Добавить**.
   В конструкторе ничего не меняйте, нажмите **Готово** — откроется форма.
3. Внизу окна формы перейдите на вкладку **Команды** (справа от «Реквизиты»).
   Добавьте две команды кнопкой «Добавить»:
   - `ПроверитьСвязь` — в свойствах в поле «Действие» нажмите лупу,
     выберите «Создать на клиенте» → откроется модуль;
   - `ЗагрузитьЗаказы` — то же самое.
4. Перетащите обе команды из списка на форму — появятся кнопки.
5. Откройте **модуль формы** (вкладка «Модуль» внизу), удалите всё, что там есть,
   и вставьте целиком содержимое файла `МодульФормыОбработки.bsl`.
6. В начале модуля в функции `ПолучитьНастройки()` заполните:
   - `Сервер` — домен сайта без https:// (например `dc-bakery.kz`);
   - `Токен` — строка из шага 0;
   - `ИмяОрганизации` — часть названия вашей организации в 1С
     (можно оставить пустым, возьмётся первая).
7. **Файл → Сохранить** → сохраните как `ЗагрузкаЗаказовDCBakery.epf`
   (например, на рабочий стол).

## Шаг 2. Запуск в 1С

1. Откройте 1С обычным образом (режим «Предприятие»).
2. Меню **Файл → Открыть…** → выберите `ЗагрузкаЗаказовDCBakery.epf`.
   Если 1С спросит про «опасные действия» — нажмите «Разрешить».
3. Нажмите **Проверить связь** — должно показать «Связь есть. Заказов за период: N».
4. Нажмите **Загрузить заказы** — обработка создаст контрагентов (в группе
   «B2B сайт») и счета (непроведённые, с пометкой «Сайт DC Bakery, заказ №…»).
   Бухгалтер открывает счёт, проверяет и проводит.

Повторное нажатие безопасно: заказы, по которым счёт уже создан, пропускаются.

## Постоянное место (по желанию)

Чтобы не открывать файл каждый раз: **Администрирование → Печатные формы,
отчёты и обработки → Дополнительные отчёты и обработки** → включить галочку →
«Добавить из файла» → выбрать .epf и указать раздел, где показывать команду.

## Если что-то не так

- «ОШИБКА СВЯЗИ … 401» — токен в модуле не совпадает с `ONEC_EXPORT_TOKEN` на сайте.
- «ОШИБКА СВЯЗИ … 503» — токен не задан на Render.
- Ошибка про сертификат/соединение — у сервера 1С нет доступа в интернет;
  тогда запустите обработку в файловом режиме на компьютере с интернетом
  или напишите нам — переключим запрос на клиентскую сторону.
- Любая другая ошибка при загрузке — пришлите скриншот сообщения целиком:
  обработка написана «вслепую» под типовую конфигурацию, 1–2 правки под ваш
  релиз — нормальная часть запуска.

Загружаются заказы за последние 14 дней (кроме отменённых и неподтверждённых).
Период можно расширить: в модуле `Путь` добавьте `?from=2026-07-01&to=2026-07-31`.
